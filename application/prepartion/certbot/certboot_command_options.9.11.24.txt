Certbot supports a lot of command line options. Here’s the full list, from certbot --help all:
Cert bot поддерживает множество опций командной строки. Вот полный список из certbot --help:

usage: 
использование:
  certbot [SUBCOMMAND] [options] [-d DOMAIN] [-d DOMAIN] ...

Certbot can obtain and install HTTPS/TLS/SSL certificates.  By default,
it will attempt to use a webserver both for obtaining and installing the
certificate. The most common SUBCOMMANDS and flags are:

Certbot может получать и устанавливать сертификаты HTTPS/TLS/SSL.  По умолчанию
он будет пытаться использовать веб-сервер как для получения, так и для установки
сертификата. Наиболее распространенными ПОДКОММАНДАМИ и флагами являются:

obtain, install, and renew certificates:
получение, установка и обновление сертификатов:
    (default) run   Obtain & install a certificate in your current webserver
                    Получите и установите сертификат на вашем текущем веб-сервере
    certonly        Obtain or renew a certificate, but do not install it
                    Получите или обновите сертификат, но не устанавливайте его
    renew           Renew all previously obtained certificates that are near expiry
                    Обновите все ранее полученные сертификаты, срок действия которых подходит к концу
    enhance         Add security enhancements to your existing configuration
                    Добавьте улучшения безопасности в существующую конфигурацию
   -d DOMAINS       Comma-separated list of domains to obtain a certificate for
                    Список доменов, разделенных запятыми, для которых необходимо получить сертификат

  --apache          Use the Apache plugin for authentication & installation
                    Используйте плагин Apache для аутентификации и установки
  --standalone      Run a standalone webserver for authentication
                    Запустите автономный веб-сервер для проверки подлинности
  --nginx           Use the Nginx plugin for authentication & installation
                    Используйте плагин Nginx для аутентификации и установки
  --webroot         Place files in a server's webroot folder for authentication
                    Поместите файлы в корневую папку сервера для проверки подлинности
  --manual          Obtain certificates interactively, or using shell script hooks
                    Получение сертификатов в интерактивном режиме или с помощью перехватчиков командной строки

   -n               Run non-interactively
  --test-cert       Obtain a test certificate from a staging server
                    Запуск в неинтерактивном режиме
  --dry-run         Test "renew" or "certonly" without saving any certificates to disk
                    Протестируйте "обновить" или "certonly" без сохранения каких-либо сертификатов на диске

manage certificates:
управление сертификатами:
    certificates    Display information about certificates you have from Certbot
                    Отображать информацию о сертификатах, которые вы получили от Certbot
    revoke          Revoke a certificate (supply --cert-name or --cert-path)
                    Отозвать сертификат (указать --cert-name или --certpath)
    delete          Delete a certificate (supply --cert-name)
                    Удалить сертификат (указать --cert-name)
    reconfigure     Update a certificate's configuration (supply --cert-name)
                    Обновите конфигурацию сертификата (укажите --cert-name)

manage your account:
управляйте своей учетной записью:
    register        Create an ACME account
                    Создайте учетную запись ACME
    unregister      Deactivate an ACME account
                    Деактивировать учетную запись ACME
    update_account  Update an ACME account
                    Обновите учетную запись ACME
    show_account    Display account details
                    Отображение сведений об учетной записи
  --agree-tos       Agree to the ACME server's Subscriber Agreement
                    Соглашаетесь с подписным соглашением ACME server
   -m EMAIL         Email address for important account notifications
                    Адрес электронной почты для получения важных уведомлений об учетной записи

При создании нового сертификата укажите имя нового сертификата. (по умолчанию: первое указанное домен или имя существующего сертификата в вашей системе для тех же доменов)

Это может спровоцировать веб-сервер  ерезагружается для временного изменения и отката файлов конфигурации. --команды pre-hook и --post-hook выполняются по умолчанию. -команды -deploy-hook не выполняют запускайте, если только это не включено с помощью --run-deploy-hooks. Тест

                        После настройки запросов дождитесь ввода данных пользователем

automation:
автоматизация:
  Flags for automating execution & other tweaks
  Флаги для автоматизации выполнения и другие настройки

  --keep-until-expiring, --keep, --reinstall
                        If the requested certificate matches an existing
                        certificate, always keep the existing one until it is
                        due for renewal (for the 'run' subcommand this means
                        reinstall the existing certificate). (default: Ask)
                        Если запрашиваемый сертификат соответствует существующему
                        сертификат, всегда сохраняйте существующий до тех пор, пока не наступит срок его обновления (для подкоманды "выполнить" это означает переустановите существующий сертификат). (по умолчанию: Спросить)
  --expand              If an existing certificate is a strict subset of the
                        requested names, always expand and replace it with the
                        additional names. (default: Ask)
                        Если существующий сертификат является строгим подмножеством запрошенных имен, всегда расширяйте и заменяйте его дополнительными именами. (по умолчанию: Спрашивать)
  --version             show program's version number and exit
                        покажите номер версии программы и завершите работу
  --force-renewal, --renew-by-default
                        If a certificate already exists for the requested
                        domains, renew it now, regardless of whether it is
                        near expiry. (Often --keep-until-expiring is more
                        appropriate). Also implies --expand. (default: False)
                        Если сертификат для запрошенных доменов уже существует, обновите его сейчас, независимо от того, подходит ли срок его действия.
                        (Часто более уместно использовать параметр сохранять до истечения срока действия). Также подразумевается параметр расширить. (по умолчанию: False)
  --renew-with-new-domains
                        If a certificate already exists for the requested
                        certificate name but does not match the requested
                        domains, renew it now, regardless of whether it is
                        near expiry. (default: False)
                        Если сертификат уже существует для запрошенного
                        имя сертификата, но не соответствует запрошенному
                        домены, продлевайте его сейчас, независимо от того, подходит ли срок его действия к концу. (по умолчанию: False)
  --reuse-key           When renewing, use the same private key as the
                        existing certificate. (default: False)
                        При обновлении используйте тот же закрытый ключ, что и для существующего сертификата. (по умолчанию: False)
  --no-reuse-key        When renewing, do not use the same private key as the
                        existing certificate. Not reusing private keys is the
                        default behavior of Certbot. This option may be used
                        to unset --reuse-key on an existing certificate.
                        (default: False)
                        При обновлении не используйте тот же закрытый ключ, что и в существующем сертификате. По умолчанию Certbot не использует закрытые ключи повторно. Эта опция может использоваться для отмены установки --reuse-key для существующего сертификата. (по умолчанию: False)
  --new-key             When renewing or replacing a certificate, generate a
                        new private key, even if --reuse-key is set on the
                        existing certificate. Combining --new-key and --reuse-
                        key will result in the private key being replaced and
                        then reused in future renewals. (default: False)
                        При обновлении или замене сертификата создайте новый закрытый ключ, аже если для существующего сертификата установлен --reuse-key.  бъединение --нового-ключа и --повторного использования- использование ключа приведет к замене закрытого ключа и последующему повторному использованию при будущих обновлениях. (по умолчанию:  alse)
  --allow-subset-of-names
                        When performing domain validation, do not consider it
                        a failure if authorizations can not be obtained for a
                        strict subset of the requested domains. This may be
                        useful for allowing renewals for multiple domains to
                        succeed even if some domains no longer point at this
                        system. This option cannot be used with --csr.
                        (default: False)
                        При выполнении проверки домена не учитывайте это сбой, если не удается получить разрешения на строгое подмножество запрашиваемых оменов. Это может быть полезно для успешного обновления нескольких доменов, даже если некоторые домены больше не указывают на это система. Этот параметр нельзя использовать с --csr. (по умолчанию: False)
  --agree-tos           Agree to the ACME Subscriber Agreement (default: Ask)
                        Соглашаться с подписным соглашением ACME (по умолчанию: Спрашивать)
  --duplicate           Allow making a certificate lineage that duplicates an
                        existing one (both can be renewed in parallel)
                        (default: False)
                        Разрешить создание сертификата, дублирующего существующий (оба сертификата могут обновляться параллельно) (по умолчанию: False)
  -q, --quiet           Silence all output except errors. Useful for
                        automation via cron. Implies --non-interactive.
                        (default: False)
                        Отключение всех выходных данных, кроме ошибок. Полезно для автоматизации с помощью cron. Подразумевает -неинтерактивный. (по умолчанию: False)

  --staple-ocsp         Enables OCSP Stapling. A valid OCSP response is

renew:
возобновлять:
  The 'renew' subcommand will attempt to renew any certificates previously
  obtained if they are close to expiry, and print a summary of the results.
  By default, 'renew' will reuse the plugins and options used to obtain or
  most recently renew each certificate. You can test whether future renewals
  will succeed with `--dry-run`. Individual certificates can be renewed with
  the `--cert-name` option. Hooks are available to run commands before and
  after renewal; see https://certbot.eff.org/docs/using.html#renewal for
  more information on these.
  Подкоманда "обновить" попытается обновить все ранее полученные сертификаты, если срок их действия близок, и напечатает сводку результатов. По умолчанию "обновить" будет повторно использовать плагины и опции, использованные для получения или последнего обновления каждого сертификата. Вы можете проверить, будут ли будущие обновления доступны. завершится успешно с помощью "--dry-run". Отдельные сертификаты можно обновить с помощью опции "--cert-name". Доступны перехватчики для запуска команд до и после обновления; смотрите раздел https://certbot.eff.org/docs/using.html#renewal для получения дополнительной информации о них.

  --pre-hook PRE_HOOK   Command to be run in a shell before obtaining any
                        certificates. Unless --disable-hook-validation is
                        used, the command’s first word must be the absolute
                        pathname of an executable or one found via the PATH
                        environment variable. Intended primarily for renewal,
                        where it can be used to temporarily shut down a
                        webserver that might conflict with the standalone
                        plugin. This will only be called if a certificate is
                        actually to be obtained/renewed. When renewing several
                        certificates that have identical pre-hooks, only the
                        first will be executed. (default: None)
                        Команда, которая должна быть запущена в командной строке перед получением каких-либо сертификатов. Если не используется параметр --disable-hook-validation, первое слово команды должно быть абсолютным путь к исполняемому файлу или имя, найденное с помощью переменной окружения PATH. Предназначен в первую очередь для обновления, где его можно использовать для временного выключения веб-сервера, который может конфликтовать с автономным плагин. manual:
  Authenticate through manual configuration or custom shell scripts. When
  using shell scripts, an authenticator script must be provided. The
  environment variables available to this script depend on the type of
  challenge. $CERTBOT_DOMAIN will always contain the domain being
  authenticated. For HTTP-01 and DNS-01, $CERTBOT_VALIDATION is the
  validation string, and $CERTBOT_TOKEN is the filename of the resource
  requested when performing an HTTP-01 challenge. An additional cleanup
  script can also be provided and can use the additional variable
  $CERTBOT_AUTH_OUTPUT which contains the stdout output from the auth
  script. For both authenticator and cleanup script, on HTTP-01 and DNS-01
  challenges, $CERTBOT_REMAINING_CHALLENGES will be equal to the number of
  challenges that remain after the current one, and $CERTBOT_ALL_DOMAINS
  contains a comma-separated list of all domains that are challenged for the
  current certificate.

  Выполните аутентификацию с помощью ручной настройки или пользовательских сценариев оболочки. При использовании сценариев оболочки необходимо предоставить сценарий проверки подлинности. Переменные среды, доступные для этого сценария, зависят от типа запроса. $CERTBOT_DOMAIN всегда будет содержать домен, для которого выполняется аутентификация. Для HTTP-01 и DNS01 $CERTBOT_VALIDATION - это строка проверки, а $CERTBOT_TOKEN - это имя файла ресурса запрашиваемого при выполнении запроса HTTP-01. Также может быть предоставлен дополнительный сценарий очистки, который может использовать дополнительную переменную $CERTBOT_AUTH_OUTPUT, который содержит стандартный вывод из сценария аутентификации. Как для аутентификатора, так и для сценария очистки, при вызовах HTTP-01 и DNS-01 значение $CERTBOT_REMAINING_CHALLENGES будет равно количеству вызовов, оставшихся после текущего, и $CERTBOT_ALL_DOMAINS. содержит разделенный запятыми список всех доменов, у которых запрашивается текущий сертификат.

webroot:
  Saves the necessary validation files to a .well-known/acme-challenge/
  directory within the nominated webroot path. A separate HTTP server must
  be running and serving files from the webroot path. HTTP challenge only
  (wildcards not supported).
  Сохраняет необходимые файлы проверки в каталоге .well-known/acme-challenge/, указанном в указанном пути webroot. Должен быть запущен отдельный HTTP-сервер, обслуживающий файлы из пути webroot. Только HTTP-вызов (подстановочные знаки не поддерживаются).

  --webroot-path WEBROOT_PATH, -w WEBROOT_PATH
                        public_html / webroot path. This can be specified
                        multiple times to handle different domains; each
                        domain will have the webroot path that preceded it.
                        For instance: `-w /var/www/example -d example.com -d
                        www.example.com -w /var/www/thing -d thing.net -d
                        m.thing.net` (default: Ask)
                        путь public_html / webroot. Его можно указать несколько раз для обработки разных доменов; для каждого домена будет указан путь webroot, который ему предшествовал. Например: `-w /var/www/example -d example.com -d www.example.com -w /var/www/thing -d thing.net -d m.thing.net` (по умолчанию: Спросить)
  --webroot-map WEBROOT_MAP
                        JSON dictionary mapping domains to webroot paths; this
                        implies -d for each entry. You may need to escape this
                        from your shell. E.g.: --webroot-map
                        '{"eg1.is,m.eg1.is":"/www/eg1/", "eg2.is":"/www/eg2"}'
                        This option is merged with, but takes precedence over,
                        -w / -d entries. At present, if you put webroot-map in
                        a config file, it needs to be on a single line, like:
                        webroot-map = {"example.com":"/var/www"}. (default:
                        {})
                        Словарь JSON сопоставляет домены с путями webroot; это подразумевает -d для каждой записи. Возможно, вам придется экранировать это из вашей оболочки. Например: --webroot-map '{"eg1.is,m.eg1.is":"/www/eg1/", "eg2.is":"/www/eg2"}" Этот параметр объединен с параметрами -w / -d, но имеет приоритет над ними. В настоящее время, если вы помещаете webroot-map в конфигурационный файл, он должен быть в одной строке, например: webroot-map = {"example.com":"/var/www"}. (по умолчанию: {})
